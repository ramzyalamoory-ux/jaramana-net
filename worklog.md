## Task ID: 1 - Full Stack Developer
### Work Task
بناء نظام إدارة مشتركي الإنترنت باستخدام Next.js 15 مع Prisma و SQLite

### Work Summary
تم بناء نظام كامل لإدارة مشتركي الإنترنت يتضمن:

#### 1. قاعدة البيانات (Prisma + SQLite):
- إنشاء موديل `Subscriber` للمشتركين مع الحقول:
  - id, name, phone (unique), address, monthlyFee, isActive
- إنشاء موديل `Invoice` للفواتير مع الحقول:
  - id, subscriberId, month, amount, isPaid, dueDate, paidAt, notes
- علاقة one-to-many بين المشترك والفواتير
- unique constraint على (subscriberId, month)

#### 2. API Routes:
- `/api/subscribers` (GET, POST): جلب وإضافة المشتركين
- `/api/subscribers/[id]` (GET, PUT, DELETE): جلب وتحديث وحذف مشترك
- `/api/invoices` (GET, POST): جلب وإضافة الفواتير مع إمكانية الفلترة
- `/api/invoices/[id]` (GET, PUT): جلب وتحديث حالة الفاتورة
- `/api/check-invoice` (GET): البحث عن فاتورة مشترك برقم الهاتف

#### 3. واجهة المستخدم:
- تصميم عربي RTL باستخدام Tailwind CSS و shadcn/ui
- واجهة الزائر:
  - حقل بحث برقم الهاتف
  - عرض بيانات المشترك وحالة الاشتراك
  - قائمة الفواتير مع المبلغ المستحق
- واجهة المسؤول:
  - تبويبات للمشتركين والفواتير والإضافة
  - جدول المشتركين مع أزرار تعديل/حذف/إضافة فاتورة
  - جدول الفواتير مع إمكانية تحديد المدفوعة
  - نموذج إضافة مشترك جديد
- تصميم responsive للموبايل
- ألوان احترافية (أخضر للمدفوع، أحمر للغير مدفوع)

#### 4. الملفات المنشأة:
- `prisma/schema.prisma`: تعريف قاعدة البيانات
- `src/app/api/subscribers/route.ts`: API المشتركين
- `src/app/api/subscribers/[id]/route.ts`: API مشترك محدد
- `src/app/api/invoices/route.ts`: API الفواتير
- `src/app/api/invoices/[id]/route.ts`: API فاتورة محددة
- `src/app/api/check-invoice/route.ts`: API البحث
- `src/app/page.tsx`: الواجهة الأمامية الكاملة

#### 5. التحقق:
- تم تشغيل `npm run lint` بنجاح
- التطبيق يعمل على المنفذ 3000
- تم إنشاء قاعدة البيانات بنجاح
